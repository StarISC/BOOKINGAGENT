< SelectedOptions OptionCode =" FUEL " />

In all the above cases the item will be included in the pricing. 
 
Non-Mandatory Auto Add Charges: 
During the pricing booking process the non-mandatory charges will be retained in the 
booking unless the explicit opt out is specified. 
< SelectedOptions OptionCode =" FUEL " SelectedOptionsIndicator =" true" /> 
< SelectedOptions OptionCode =" FUEL " /> 
< ** FUEL option is not sent back in the request **  /> 
 
In all the above cases the item will be included in the pricing. 
Non-mandatory charges can be opted out by explicating setting th e 
selectedOptionIndicator to “False”. 
 
< SelectedOptions OptionCode =" FUEL " SelectedOptionsIndicator =" false " /> 
If “false” is sent the item will not be included in the pricin g response. This 
declined item can only be added back by back office process by calli ng in.

25. Cruise Confirm Booking Message 
25.1 Functional Overview 
The Cruise Confirm Booking message is used to creat e a booking, amend a booking or 
cancel a booking on the RCL reservation system for a specific sailing, with a specified 
mode of transportation/gateway city pair and curren cy, and a selected fare/category pair, 
for one or more passengers in a specific cabin. 
Prior to submitting a Confirm Booking request messa ge, a cabin must have been placed 
on hold.  The only exception to this is if a cabin is reserved as a guarantee, rather than a 
specific cabin. If a booking transaction is not com pleted when the cabin is held, it can 
still be booked if it is available at the time of t he booking request receipt. In case the 
cabin is not available, then an error is sent back to select a new cabin. 
The response message indicates if the request was s uccessful and returns a Reservation 
ID number. 
 
In amend mode, the client must provide the existing  passenger ids for the given booking. 
Please use //ContactInfo/@GuestRefNumber to provide  the existing passenger ids.

validity and availability of all the components tha t have been selected for a 
booking. If errors are found, the process stops and  the appropriate error message 
is returned in the response.

be confirmed, the transaction program will confirm the new or amended booking 
by recording the changes in the RCL reservation sys tem.

credit card payment will be applied to the booking.
